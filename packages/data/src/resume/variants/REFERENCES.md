# References

This file collects external resources that have influenced the writing harness for resume variants and cover letters.

These links are for reference purposes only. Agents should not try to mimic an article literally. Instead, they should follow `WRITING_STYLE.md`, which distills the relevant guidance into local house rules.

## Cover Letters

### Jason Fried on cover letters

Reference:

- [Cover letters? Yes.](https://world.hey.com/jason/cover-letters-yes-25dcdd0c)

Distilled takeaways adopted into this repo:

- Cover letters still matter as a first signal of effort, care, diligence, and communication ability.
- Strong cover letters feel specific to the role, not mass-produced.
- Good letters sound like the person, not a generic template.
- Written clarity matters because the letter is also a writing sample.
- The best letters are concise, readable, and worth continuing to read.
- If a letter feels interchangeable, it is usually weak.

Notes:

- Use this source to reinforce the value of clarity, effort, and specificity.
- Do not imitate the article’s voice directly.
- Prefer the repo’s own `WRITING_STYLE.md` when there is any ambiguity.

### DHH on personal, fluent cover letters

Reference:

- [DHH on X](https://x.com/dhh/status/1925440134566060170?s=46&t=ssP6EM3-WGWpWUN96NHzSA)

Quoted idea:

> A good cover letter is always the first gate for landing a job at 37signals. I don't care if we're the last company to require them, we're never going to hire someone who can't write a single, personal, fluent page on why they want THIS job.

Distilled takeaways adopted into this repo:

- A strong cover letter should feel personal, not interchangeable.
- It should show why the candidate wants this specific role.
- Fluency and written judgment are part of the evaluation, not just the content itself.
- Cover letters should act as proof of clear written communication, not merely a polite ritual.

Notes:

- Use this source to reinforce specificity, fluency, and genuine interest.
- Keep the repo style grounded in `WRITING_STYLE.md` rather than trying to mimic social-post tone.
